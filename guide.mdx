## Overview

In this guide, you'll build a real-time conversational voice agent that handles customer support calls for a fictional SaaS product called "Acme Cloud." The agent listens to user speech, transcribes it with AssemblyAI's Universal-3 Pro model, generates responses with an LLM, and speaks them back using text-to-speech — all with sub-second latency and natural turn-taking.

### What you'll use

- **AssemblyAI Universal-3 Pro** — Streaming STT with punctuation-based turn detection
- **Pipecat** — Pipeline framework for building voice agents
- **Cerebras** — LLM for generating conversational responses
- **Rime** — Text-to-speech for natural voice output

## Prerequisites

To follow along, you'll need:

- Python 3.10 or higher
- An [AssemblyAI API key](https://www.assemblyai.com/dashboard/signup)
- A [Cerebras API key](https://cloud.cerebras.ai/)
- A [Rime API key](https://rime.ai/)

## Project setup

Create a new project directory and set up a virtual environment:

```bash
mkdir voice-agent && cd voice-agent
python -m venv venv
source venv/bin/activate
```

Install the dependencies:

```bash
pip install "pipecat-ai[assemblyai,cerebras,rime,silero,daily,webrtc]" python-dotenv fastapi uvicorn pipecat-ai-small-webrtc-prebuilt
```

Download the Pipecat run helper, which provides the WebRTC transport and browser UI:

```bash
curl -O https://raw.githubusercontent.com/pipecat-ai/pipecat/9f223442c2799d22aac8a552c0af1d0ae7ff42c2/src/pipecat/examples/run.py
```

Create a `.env` file with your API keys:

```env
ASSEMBLYAI_API_KEY=your_assemblyai_api_key
CEREBRAS_API_KEY=your_cerebras_api_key
RIME_API_KEY=your_rime_api_key
```

Now create a `voice_agent.py` file and follow along below.

  This is where AssemblyAI's Universal-3 Pro model is configured for streaming transcription.

  **`vad_force_turn_endpoint=False`** is critical — it tells Pipecat to rely on AssemblyAI's punctuation-based turn detection instead of VAD-based silence detection.   - **`keyterms_prompt`** — Boosts recognition accuracy for domain-specific terms like "Pro Plan", "Acme Cloud", and "billing". These terms are appended to the model's default prompt automatically.

## Run the agent

Start the voice agent:

```bash
python voice_agent.py
```

Open [http://localhost:7860](http://localhost:7860) in your browser, click **Connect**, and start talking. The agent will greet you and help with Acme Cloud account questions.

## How turn detection works

AssemblyAI's Universal-3 Pro uses punctuation-based turn detection, which is more accurate than silence-only detection:

1. You speak, and audio streams to AssemblyAI in real time
2. When you pause for 100ms (`min_end_of_turn_silence_when_confident`), the model checks if the transcript ends with terminal punctuation (`.` `?` `!`)
3. If terminal punctuation is found, the turn ends immediately and the transcript is sent to the LLM
4. If no terminal punctuation, a partial transcript is emitted and the model continues listening
5. If silence reaches 1200ms (`max_turn_silence`), the turn is forced to end regardless

This means short pauses mid-sentence (like "I need help with... my billing") won't prematurely trigger a response, while clear sentence endings get processed immediately.

## Next steps

- **Update keyterms mid-conversation**: Use `UpdateConfiguration` to change `keyterms_prompt` as the conversation progresses (e.g., add account-specific terms after the user identifies themselves)
- **Swap the LLM**: Replace Cerebras with OpenAI, Anthropic, or any other Pipecat-supported LLM service
- **Add phone support**: Switch from WebRTC to the Twilio transport to handle real phone calls
- **Log transcripts**: Add a processor to the pipeline that captures and stores conversation transcripts
